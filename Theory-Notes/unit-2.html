Problem Solving & Search Strategies:

Problem Solving & Search Strategies is a fundamental topic in computer science and artificial intelligence (AI), focusing on how to find solutions to complex problems systematically and efficiently. Here's a breakdown of the key concepts:

1. Problem Solving in AI

Problem-solving involves:

         Defining a problem clearly

         Representing it in a structured way

         Finding a sequence of actions or steps to reach a goal

Components of a problem:

1.      Initial state where the agent starts

2.      Actions possible moves or decisions

3.      Transition model what each action does

4.      Goal test how to check if the goal is reached

5.      Path cost cost of a solution (e.g., distance, time)

2. State Space Search

A problem is often represented as a state space, where:

         Nodes represent states

         Edges represent actions

         The goal is to find a path from the start node to a goal node

3. Types of Search Strategies

A. Uninformed (Blind) Search

No information about the goal is used other than how to reach it.

1.      Breadth-First Search (BFS)

o    Explores all nodes at the current depth before going deeper

o    Guarantees the shortest path if all costs are equal

o    Time/space complexity: O(b^d)

2.      Depth-First Search (DFS)

o    Explores as deep as possible before backtracking

o    Memory-efficient but not guaranteed to find the shortest path

3.      Uniform Cost Search (UCS)

o    Expands the node with the lowest path cost

o    Optimal if costs are positive

B. Informed (Heuristic) Search

Uses additional information (heuristics) to guide the search.

1.      Greedy Best-First Search

o    Chooses the node that appears closest to the goal (based on a heuristic h(n))

o    Not guaranteed to be optimal

2.      A Search*

o    Combines cost-so-far (g(n)) + estimated cost to goal (h(n))

o    f(n) = g(n) + h(n)

o    Optimal if h(n) is admissible (never overestimates)

4. Heuristics

Heuristics are problem-specific estimates of how close a state is to the goal. Good heuristics:

         Improve search efficiency

         Must be admissible for A* to be optimal

         May be consistent (monotonic) for further performance guarantees

5. Examples of Applications

         Path finding in maps or games

         Puzzle solving (e.g., 8-puzzle)

         Robotics

         Automated planning

         Decision-making systems

Uninformed Search Strategies (Blind Search)

Uninformed search strategies explore the search space without any domain-specific knowledge they dont use heuristics or information about how close a state is to the goal. They treat all paths as equal unless otherwise specified.

1. Breadth-First Search (BFS)

         Strategy: Explores all nodes at one depth level before going deeper.

         Data structure: Queue (FIFO)

Properties:

         Complete: Yes (if the branching factor is finite)

         Optimal: Yes (if all step costs are equal)

         Time complexity: O(b^d)

         Space complexity: O(b^d)

b = branching factor, d = depth of the shallowest goal

2. Depth-First Search (DFS)

         Strategy: Explores as deep as possible along each branch before backtracking.

         Data structure: Stack (LIFO), often recursive

Properties:

         Complete: No (can get stuck in infinite loops in infinite-depth spaces)

         Optimal: No

         Time complexity: O(b^m)

         Space complexity: O(bm)

m = maximum depth of the search tree

3. Depth-Limited Search

         DFS with a depth limit l to avoid infinite paths.

         Prevents going deeper than a specified limit.

Properties:

         Complete: No (if goal is beyond depth l)

         Optimal: No

         Time complexity: O(b^l)

         Space complexity: O(bl)

4. Iterative Deepening Depth-First Search (IDDFS)

         Combines DFSs space-efficiency and BFSs completeness/optimality.

         Repeatedly applies DFS with increasing depth limits.

Properties:

         Complete: Yes

         Optimal: Yes (if all step costs are equal)

         Time complexity: O(b^d)

         Space complexity: O(bd)

5. Uniform Cost Search (UCS)

         Explores paths in order of lowest cumulative cost.

         Special case of Dijkstras algorithm.

Properties:

         Complete: Yes

         Optimal: Yes (for positive path costs)

         Time complexity: O(b^C/ε)

         Space complexity: O(b^C/ε)

C = cost of the optimal solution, ε = minimum step cost

Breadth-First Search (BFS)

Breadth-First Search (BFS) is an uninformed search algorithm that explores all the nodes at the present depth before moving on to the nodes at the next level. It is ideal when the solution is close to the root and step costs are equal.

Key Concepts

         Uses a queue (FIFO) to keep track of nodes to explore.

         Explores nodes level-by-level.

         Guarantees the shortest path in terms of number of steps (if step costs are equal).

Algorithm Steps

1.      Start with the initial node.

2.      Add the initial node to a queue.

3.      Repeat:

o    Dequeue the front node.

o    If it's the goal, return the path.

o    Else, enqueue all unvisited children of the node.

4.      If the queue is empty, return failure.

Example Problem:

Find a path from A to G in the following graph:

A

/ \

B C

/ \ \

D E F

\

G

Python Code:

from collections import deque

def bfs(graph, start, goal):

visited = set()

queue = deque([[start]]) # queue of paths

while queue:

path = queue.popleft() # FIFO

node = path[-1]

if node == goal:

return path

if node not in visited:

visited.add(node)

for neighbor in graph.get(node, []):

new_path = path + [neighbor]

queue.append(new_path)

return None # No path found

Sample Graph:

graph = {

'A': ['B', 'C'],

'B': ['D', 'E'],

'C': ['F'],

'F': ['G'],

'D': [],

'E': [],

'G': []

}

# Run the BFS

path = bfs(graph, 'A', 'G')

print("Path found:", path)

Output:

Path found: ['A', 'C', 'F', 'G']

Depth First Search (DFS):

Depth First Search (DFS) is an algorithm used to traverse or search through a graph or tree data structure. It explores as far down a branch of the graph as possible before backtracking, making it particularly useful for solving problems where you need to explore all possible paths or find solutions in deep structures.

1. Key Concepts of DFS

  • Graph Representation: DFS can be applied to both directed and undirected graphs, and can be implemented on both adjacency lists and adjacency matrices.
  • Traversal Process: Starting from a source node, the algorithm visits as far as possible down a path before backtracking.
  • Backtracking: When no more unvisited neighbors are left for the current node, the algorithm backtracks to the previous node and continues exploring the next available paths.
  • Stack: DFS uses a stack (either explicitly or through recursion) to keep track of the nodes to visit next.

2. Algorithm Steps

Heres how the DFS algorithm works:

  1. Start at a Node (usually the root or any arbitrary node).
  2. Visit the Node: Mark it as visited.
  3. Explore Neighbors: For each unvisited neighboring node, perform DFS recursively.
  4. Backtrack: Once all neighbors are visited, backtrack to the previous node.
  5. Repeat until all reachable nodes have been visited.

3. DFS Algorithm (Recursive Version)

def dfs(graph, node, visited=None):

if visited is None:

visited = set() # Initialize an empty set to store visited nodes

visited.add(node) # Mark the current node as visited

print(node) # Optionally print or process the node

for neighbor in graph[node]: # Explore neighbors of the current node

if neighbor not in visited:

dfs(graph, neighbor, visited) # Recursively visit unvisited neighbors

return visited

Example: DFS on a Graph

Given a graph represented by an adjacency list:

graph = {

'A': ['B', 'C'],

'B': ['A', 'D', 'E'],

'C': ['A', 'F'],

'D': ['B'],

'E': ['B', 'F'],

'F': ['C', 'E']

}

Calling dfs(graph, 'A') would traverse the graph in the following order (one possible path):

A → B → D → E → F → C

4. DFS Using Stack (Iterative Version)

An iterative version of DFS uses an explicit stack to simulate the recursion stack.

def dfs_iterative(graph, start):

visited = set() # Set to keep track of visited nodes

stack = [start] # Stack for DFS

 

while stack:

node = stack.pop() # Pop a node from the stack

if node not in visited:

visited.add(node) # Mark as visited

print(node) # Optionally print or process the node

# Push all unvisited neighbors to the stack

stack.extend(neighbor for neighbor in graph[node] if neighbor not in visited)

return visited

5. Time Complexity

  • Time Complexity: O(V + E)
    • V: Number of vertices (nodes).
    • E: Number of edges.

DFS visits each node and edge once, so the time complexity depends on both the number of nodes and edges in the graph.

  • Space Complexity: O(V)
    • The space complexity is mainly due to the storage of the visited set and the recursion stack (or the explicit stack in the iterative version).

6. Applications of DFS

DFS is useful in various applications, including:

  • Pathfinding: Finding a path between two nodes.
  • Topological Sorting: For directed acyclic graphs (DAGs), DFS can help in ordering nodes.
  • Cycle Detection: Detecting cycles in graphs (important in directed graphs).
  • Solving Puzzles: DFS can explore all possible configurations in problems like mazes, Sudoku, and the N-Queens problem.
  • Connected Components: Finding all connected components in an undirected graph.
  • Backtracking: DFS is often used in backtracking algorithms (e.g., solving a maze, solving the N-Queens problem).

Iterative Deepening Depth-First Search (IDDFS)

Iterative Deepening Depth-First Search (IDDFS) is a graph traversal algorithm that combines the advantages of Depth First Search (DFS) and Breadth First Search (BFS). It iteratively performs DFS with increasing depth limits, ensuring that all nodes are explored while avoiding the excessive memory usage of BFS.

1. Key Concepts of IDDFS

  • Combination of DFS and BFS: IDDFS is essentially a DFS where the depth limit is increased step-by-step, mimicking BFS behavior in terms of layer-by-layer exploration.
  • Depth Limiting: In each iteration, DFS is performed with a different depth limit. The depth limit increases gradually from 0 to the maximum depth of the tree or graph.
  • Memory Efficiency: IDDFS uses the memory of DFS (which is O(d), where d is the depth of the tree) while still ensuring that all nodes are visited in a breadth-first manner.

2. IDDFS Algorithm

  1. Start with depth limit 0.
  2. Perform DFS up to the current depth limit.
  3. Increase the depth limit and repeat DFS until the goal is found or the entire search space is explored.

The key advantage of IDDFS is that it ensures that the search space is explored in order of increasing depth, which can be useful when the solution is expected to be found in shallow depths but the search space is large.

3. Algorithm Steps

  1. Set the initial depth limit to 0.
  2. Perform Depth First Search with the current depth limit.
    • If the goal is found at this depth, return the result.
    • If the limit is reached, backtrack.
  3. Increment the depth limit by 1.
  4. Repeat until the goal is found or the maximum depth is reached.

4. IDDFS Pseudocode

def dfs_with_limit(graph, start, goal, depth_limit):

# Perform a DFS with depth limit

if start == goal:

return [start]

if depth_limit <= 0:

return None

for neighbor in graph[start]:

result = dfs_with_limit(graph, neighbor, goal, depth_limit - 1)

if result:

return [start] + result

return None

 

def iddfs(graph, start, goal):

depth = 0

while True:

result = dfs_with_limit(graph, start, goal, depth)

if result:

return result

depth += 1

Example: IDDFS on a Graph

Consider the following simple graph:

A

/ \

B C

/ \ \

D E F

  • Start from node A.
  • Perform DFS with depth limit 0 (only visit A).
  • Perform DFS with depth limit 1 (visit A, then B and C).
  • Perform DFS with depth limit 2 (visit A, B, C, D, E, and F).

5. Time Complexity

  • Time Complexity: O(b^d)
    • b: Branching factor (average number of child nodes per node).
    • d: Depth of the shallowest solution.

Even though IDDFS performs multiple depth-limited DFS traversals, the time complexity is still O(b^d), as each node is visited a constant number of times.

  • Space Complexity: O(b*d)
    • The space complexity is proportional to the maximum depth and the branching factor, as it only needs to store a single path from the root to the current node in memory at any point (like DFS).

6. Advantages of IDDFS

  1. Memory Efficient: IDDFS uses the same amount of memory as DFS, which is proportional to the depth of the tree (O(d)).
  2. Optimal for Shallow Solutions: In cases where the solution is close to the root or shallow, IDDFS performs similarly to BFS, but without needing the huge memory overhead of BFS.
  3. Finds the Shortest Path: Since IDDFS explores all nodes at depth d before it explores any node at depth d+1, it guarantees that the first time a goal node is found, it is at the minimum depth.
  4. Complete: It guarantees that the solution will be found if it exists, as it explores all possible nodes.

7. Disadvantages of IDDFS

  1. Repeated Work: IDDFS repeatedly explores the same nodes at different depths. This results in more work compared to BFS, which doesn't revisit nodes once explored.
  2. Slower for Deep Solutions: If the solution is deep, IDDFS can be slower than BFS because it explores every node multiple times as it increases the depth limit.

8. Applications of IDDFS

  • Search Algorithms: IDDFS is commonly used in pathfinding problems (e.g., finding a path in a maze) or game AI where memory is a concern.
  • Puzzle Solving: It's effective in problems like the 8-puzzle or N-Queens problem, where the goal is to find a solution with the minimum depth.
  • Artificial Intelligence: IDDFS is used in game trees (like chess), where the tree depth increases with each move.

Informed Search Strategies (Heuristic Search)

Informed search strategies are search algorithms that use heuristics (problem-specific knowledge) to guide the search process more efficiently toward the goal. Unlike uninformed search algorithms (like BFS or DFS), which explore the search space blindly, informed search algorithms use extra information to make decisions that can reduce the search time and the space complexity.

1. What is a Heuristic?

         A heuristic is a problem-specific function that estimates how close a state is to the goal state. It helps the search algorithm prioritize which paths to explore first.

         A heuristic does not guarantee an optimal solution but provides a way to make the search more efficient by guiding the search in the right direction.

         Heuristics are typically represented as a function h(n), which estimates the cost or distance from the current node n to the goal node.

2. Types of Informed Search Strategies

Here are the most common informed search strategies:

A. Best-First Search (BFS)

         Goal: Expand the most promising node based on the heuristic function h(n).

         How It Works:

o    Best-First Search uses a priority queue (often implemented as a min-heap) to always expand the node with the lowest heuristic value (the most promising node).

o    It does not consider the cost to reach a node (g(n)), just the estimated cost to reach the goal (h(n)).

         Time Complexity: O(b^d) where b is the branching factor and d is the depth of the solution.

         Use Case: Often used in pathfinding problems where we want to find the most promising route first.

         Example: The Greedy Best-First Search is a specific type of Best-First Search where the heuristic is the only consideration.

B. A Search Algorithm*

         Goal: Find the shortest path from the start node to the goal node by considering both the cost to get to the node (g(n)) and the estimated cost to the goal (h(n)).

         How It Works:

o    A* Search uses the function f(n) = g(n) + h(n) to prioritize nodes. g(n) is the actual cost to reach the node from the start, and h(n) is the heuristic estimate of the cost from that node to the goal.

o    A* is both optimal and complete, meaning it will always find the optimal solution if one exists, provided the heuristic is admissible (it never overestimates the cost to reach the goal).

         Time Complexity: O(b^d) in the worst case, but A* performs much better in practice because it reduces the number of explored nodes compared to BFS.

         Example: Navigation systems like Google Maps use A* to find the shortest route.

         Admissibility: A heuristic is admissible if h(n) ≤ h*(n) for every node n, where h*(n) is the true cost to the goal from n.

C. Greedy Best-First Search

         Goal: Expand the node that seems to be the most promising based solely on the heuristic h(n), without considering the cost to reach that node.

         How It Works:

o    Greedy Best-First Search selects nodes based only on the heuristic function h(n)the node that is closest to the goal according to the heuristic is expanded next.

o    Its not guaranteed to find the shortest path, but it can be faster than other algorithms.

         Time Complexity: Similar to Best-First Search, its O(b^d) where b is the branching factor and d is the depth of the goal node.

         Use Case: Suitable for problems where you are willing to sacrifice optimality for faster performance, such as real-time navigation where the goal is to find a good enough solution quickly.

D. Iterative Deepening A (IDA)**

         Goal: Combine the space efficiency of Iterative Deepening Search (IDS) with the optimality of A Search*.

         How It Works:

o    IDA* uses the idea of depth-first search but with an iterative deepening approach. At each iteration, it deepens the search limit to f(n) = g(n) + h(n), and performs a depth-first search (DFS) up to that limit.

o    It repeatedly increases the limit until the goal is found.

o    Its space-efficient, as it uses only the memory for a single path (like DFS), but still uses f(n) to make the search more directed (like A*).

         Use Case: Suitable for problems where memory is a concern, but you still need the guarantees of optimality provided by A*.

4. Characteristics of Heuristics

  • Admissibility: A heuristic is admissible if it never overestimates the actual cost to the goal. This is important for A* to guarantee optimality.
  • Consistency (Monotonicity): A heuristic is consistent (or monotonic) if for every node n and its successor n', the estimated cost to the goal from n is less than or equal to the step cost from n to n' plus the estimated cost to the goal from n'. Consistent heuristics are always admissible.

5. Applications of Informed Search Strategies

  1. Pathfinding Algorithms: Used in GPS navigation systems, robotics, and video games for finding the shortest route or optimal path.
  2. Puzzle Solving: Solving problems like the 8-puzzle, N-Queens, and Sliding Puzzle where you need to find a sequence of moves to reach the goal configuration.
  3. Game AI: Used in algorithms like minimax with alpha-beta pruning to guide decisions based on the possible future states in chess, checkers, and other board games.
  4. Optimization Problems: Solving problems like the Traveling Salesman Problem (TSP) and knapsack problem using heuristic search.
  5. Robotics: Used for real-time path planning and movement optimization for robots.

6. Summary of Informed Search Strategies

  • A Search* is the most commonly used strategy when the goal is to find an optimal solution with both accuracy and efficiency.
  • Greedy Best-First Search can be useful when you prioritize speed over optimality, sacrificing accuracy for faster solutions.
  • Best-First Search and IDA* combine concepts of both DFS and BFS to make the search more efficient while still trying to be exhaustive.

Heuristics and Optimization:

Heuristics and Optimization are two fundamental concepts in problem-solving, especially in fields like computer science, operations research, artificial intelligence, and engineering. They are often used together to efficiently find solutions to complex problems, particularly when exact methods are too slow or impractical.

Heuristics:

A heuristic is a practical method or rule-of-thumb used to make decisions or solve problems faster when classic methods are too slow or complicated.

Key Features:

         Not guaranteed to find the optimal solution.

         Trade accuracy for speed or simplicity.

         Often problem-specific.

         Useful in NP-hard problems (e.g., Traveling Salesman, Knapsack, Scheduling).

Examples:

         Greedy algorithms (e.g., always take the best-looking step now).

         Nearest neighbor in pathfinding.

         A* algorithm with an admissible heuristic.

         Simulated Annealing and Tabu Search use heuristics to escape local optima.

Optimization:

Optimization is the process of finding the best solution from a set of feasible solutions according to some criteria.

🔑 Key Features:

         Goal: maximize or minimize an objective function.

         May be exact or approximate.

         Can be linear, nonlinear, discrete, convex, etc.

Examples:

         Linear Programming (LP) e.g., maximize profit given resource constraints.

         Integer Programming (IP) variables must be whole numbers.

         Gradient Descent used in machine learning.

         Evolutionary algorithms inspired by natural selection.

Hill Climbing:

Hill Climbing is a heuristic search algorithm used for mathematical optimization problems. It's one of the simplest forms of local search, where the goal is to find the best solution by iteratively improving a single solution based on some evaluation function.

What Is Hill Climbing?

Hill climbing is like climbing a hill where:

  • Each point on the hill represents a possible solution.
  • The elevation at a point is the quality (fitness or objective value) of that solution.
  • The goal is to reach the highest point (maximize the objective function).

How It Works:

  1. Start with an initial solution (random or heuristic-based).
  2. Evaluate neighboring solutions.
  3. If a neighbor is better, move to that neighbor.
  4. Repeat until no better neighbor exists.
  5. Output the current solution as the result.

Basic Hill Climbing Algorithm (Pseudocode):

function HillClimbing(problem):

current ← initial_solution(problem)

loop do:

neighbor ← best_neighbor(current)

if neighbor.value ≤ current.value:

return current

current ← neighbor

Variants of Hill Climbing:

  1. Steepest-Ascent Hill Climbing
    • Examines all neighbors and picks the best one.
    • More thorough, but slower.
  2. Stochastic Hill Climbing
    • Picks a random better neighbor (not necessarily the best).
    • Helps avoid some local optima.
  3. First-Choice Hill Climbing
    • Picks the first better neighbor found.
    • Faster and often good enough.
  4. Random-Restart Hill Climbing
    • Runs hill climbing multiple times from different random starting points.
    • Helps escape global traps by retrying.

Example Use Case: Traveling Salesman Problem (TSP)

  • Each solution = a specific route.
  • Neighbors = routes created by swapping two cities.
  • Fitness = total distance (lower is better).
  • Hill climbing improves the route by trying local swaps.

Simple Python Example:

import random

def hill_climb(objective, neighbors, start):

current = start

while True:

neighbor = max(neighbors(current), key=objective)

if objective(neighbor) <= objective(current):

return current

current = neighbor

 

# Example: Maximize f(x) = -x^2 + 5

def objective(x):

return -x**2 + 5

 

def neighbors(x):

step = 0.1

return [x + step, x - step]

result = hill_climb(objective, neighbors, start=0)

print("Best solution:", result)

When to Use Hill Climbing:

  • The search space is well-behaved (no large plateaus or deceptive local maxima).
  • Fast approximate solutions are acceptable.
  • Memory is limited.

Simulated Annealing:

Simulated Annealing (SA) is an optimization algorithm inspired by the annealing process in metallurgy. Its useful for finding approximate solutions to complex optimization problems, especially when the search space is large or has many local minima.

Real-World Analogy:

In metallurgy, annealing involves heating a material and then slowly cooling it to remove defects this allows atoms to settle into a more stable, low-energy configuration. Simulated Annealing uses this concept to cool down toward an optimal solution.

Key Concepts

         Solution space: All possible solutions to the problem.

         Current solution: Where you are now.

         Neighbor solution: A slightly modified version of the current one.

         Temperature (T): Controls the probability of accepting worse solutions to escape local optima.

         Cooling schedule: Strategy to reduce temperature over time.

Simulated Annealing Algorithm (Steps)

1.      Start with an initial solution.

2.      Set a high initial temperature.

3.      Repeat:

o    Generate a neighbor solution.

o    Calculate the difference in cost (ΔE).

o    If the new solution is better, accept it.

o    If worse, accept it with probability e^(-ΔE / T).

o    Lower the temperature using a cooling schedule.

4.      Stop when the system has cooled (temperature is low or max iterations reached).

Python Example: Minimize f(x) = x

import math

import random

def objective_function(x):

return x ** 2 # Goal: minimize this

def simulated_annealing(initial_temp, cooling_rate, max_iter):

# Start with a random solution

current_x = random.uniform(-10, 10)

current_energy = objective_function(current_x)

best_x = current_x

best_energy = current_energy

temperature = initial_temp

for i in range(max_iter):

# Generate a new candidate solution

new_x = current_x + random.uniform(-1, 1)

new_energy = objective_function(new_x)

# Difference in energy

delta_energy = new_energy - current_energy

# Decide whether to accept the new solution

if delta_energy < 0 or random.random() < math.exp(-delta_energy / temperature):

current_x = new_x

current_energy = new_energy

# Update best found so far

if new_energy < best_energy:

best_x = new_x

best_energy = new_energy

# Cool down

temperature *= cooling_rate

return best_x, best_energy

# Run the algorithm

best_solution, best_value = simulated_annealing(

initial_temp=1000,

cooling_rate=0.95,

max_iter=1000

)

print("Best solution found:", best_solution)

print("Value of function at best solution:", best_value)

Value of function at best solution: 1.56e-08

Constraint Satisfaction Problems (CSPs)

A Constraint Satisfaction Problem (CSP) is a mathematical framework used to model problems where the goal is to assign values to variables in a way that satisfies a set of constraints.

Structure of a CSP

A CSP is defined by:

Component

Description

Variables

A set of variables X={X1,X2,...,Xn}

Domains

A domain Di for each variable Xi​, which is the set of possible values

Constraints

A set of rules that restrict the combinations of values that variables can take

Example: Map Coloring Problem

Color each region of a map such that no adjacent regions have the same color.

         Variables: Regions (e.g., WA, NT, SA, Q)

         Domains: Colors (e.g., Red, Green, Blue)

         Constraints: No two neighboring regions can be the same color

Solving CSPs

CSPs are commonly solved using search algorithms and inference techniques:

1. Backtracking Search

A depth-first search that assigns values to variables and backtracks when a constraint is violated.

2. Forward Checking

When a variable is assigned, eliminate inconsistent values from neighboring variables' domains.

3. Constraint Propagation (Arc Consistency)

Maintains consistency by ensuring all binary constraints are satisfied (e.g., using AC-3 algorithm).

4. Heuristics

         Minimum Remaining Values (MRV): Choose the variable with the fewest legal values.

         Degree Heuristic: Choose the variable involved in the most constraints.

         Least Constraining Value (LCV): Choose the value that rules out the fewest options for neighboring variables.

return 0

Min-max Algorithm:

The Min-Max algorithm (or Minimax) is a decision-making algorithm used in two-player, turn-based, zero-sum games, like chess, tic-tac-toe, or checkers. It helps a player determine the best move to make by simulating all possible moves and counter-moves, assuming the opponent plays optimally.

Basic Idea

  • Minimax tries to maximize the player's score while assuming the opponent will minimize it.
  • The player is called the "maximizer", and the opponent is the "minimizer".
  • The algorithm explores the game tree, evaluating the final outcomes (leaf nodes), then propagates values upward to make the optimal choice.

Assumptions

  • Perfect information (both players know everything about the game state).
  • Both players play optimally.

How It Works (Step-by-Step)

  1. Build the game tree from the current state to possible future states (up to a depth or end of game).
  2. Evaluate the terminal states with a utility function (e.g., +1 for win, 0 for draw, -1 for loss).
  3. Backpropagate values:
    • At maximizer nodes, choose the maximum value.
    • At minimizer nodes, choose the minimum value.
  4. Choose the move that leads to the best value for the current player.

Example (Tic-Tac-Toe Mini Tree)

MAX

/ \

MIN MIN

/ \ / \

0 1 -1 0

The minimizer will pick the smallest value from children.

The maximizer picks the best among the minimizer's worst:

         Left MIN: min(0,1) = 0

         Right MIN: min(-1,0) = -1

         MAX picks max(0, -1) = 0

def minimax(node, depth, isMaximizingPlayer):

if node is a terminal node or depth == 0:

return evaluate(node)

if isMaximizingPlayer:

maxEval = -infinity

for child in node.children:

eval = minimax(child, depth - 1, False)

maxEval = max(maxEval, eval)

return maxEval

else:

minEval = +infinity

for child in node.children:

eval = minimax(child, depth - 1, True)

minEval = min(minEval, eval)

return minEval

Optimizations

  • Alpha-Beta Pruning: Skips unnecessary branches to improve efficiency.
  • Heuristic Evaluation: Used when the game tree is too large to reach terminal states.

Use Cases

  • Tic-Tac-Toe
  • Chess (with modifications + heuristics)
  • Connect Four
  • Any deterministic, turn-based, zero-sum game.

Alpha-Beta Pruning:

Alpha-Beta Pruning is an optimization technique for the Minimax algorithm that cuts off parts of the search tree that don't need to be explored saving time without affecting the result.

Core Idea

In Minimax, we search all possible game states to find the best move. But in many cases, we can determine early that some branches wont affect the final decision. Alpha-Beta Pruning skips those branches.

Terminology

  • Alpha (α): Best value that the maximizer can guarantee so far.
  • Beta (β): Best value that the minimizer can guarantee so far.

As the tree is searched:

  • If at any point α ≥ β, we prune that branch (no need to explore further).

How It Works

We pass down alpha and beta values while traversing:

  • Maximizing player tries to maximize alpha.
  • Minimizing player tries to minimize beta.

#Alpha-Beta Pruning Pseudocode

def alpha_beta(node, depth, alpha, beta, isMaximizingPlayer):

if node is terminal or depth == 0:

return evaluate(node)

if isMaximizingPlayer:

maxEval = -infinity

for child in node.children:

eval = alpha_beta(child, depth - 1, alpha, beta, False)

maxEval = max(maxEval, eval)

alpha = max(alpha, eval)

if beta <= alpha:

break # 🪓 Beta Cutoff

return maxEval

else:

minEval = +infinity

for child in node.children:

eval = alpha_beta(child, depth - 1, alpha, beta, True)

minEval = min(minEval, eval)

beta = min(beta, eval)

if beta <= alpha:

break # Alpha Cutoff

return minEval

Visualization of Pruning

Imagine this game tree:

MAX

/ \

MIN MIN

/ | \ / \

2 1 3 5 0

Without pruning: all nodes are evaluated.

With pruning:

  • As soon as a MIN node finds a value less than or equal to an already better option for MAX, it stops searching.

Benefits

  • Reduces time complexity:
    • Minimax: O(b^d)
    • Alpha-Beta: O(b^(d/2)) in the best case (with perfect move ordering)
  • Allows deeper searches in the same amount of time.

Real-World Use

Used in:

  • Chess engines
  • Checkers / Connect Four
  • AI bots for turn-based games

Stochastic Observable Games Overview

Stochastic Observable Games are a class of games in game theory that combine elements of:

1.      Stochastic games where the game involves randomness or probabilistic transitions between states.

2.      Observability where players may have partial, noisy, or imperfect information about the game state or opponents' actions.

Breakdown of Terms

1. Stochastic Game (also called Markov Games)

         Introduced by Lloyd Shapley in 1953.

         Played over a series of stages.

         At each stage:

o    The game is in some state.

o    Each player chooses an action.

o    A payoff is received.

o    The game transitions probabilistically to the next state.

         The transition depends on the current state and players' actions.

2. Observable / Observability

         Perfect observability: All players know the exact state and actions of others.

         Partial observability: Players only have limited or probabilistic information about the state or actions.

         Related to Partially Observable Markov Decision Processes (POMDPs) in single-agent settings.

Key Features of Stochastic Observable Games

Feature

Description

Players

Multiple (at least 2), each with their own actions and goals.

States

Game has various states, transitions between them are probabilistic.

Actions

Players choose actions that affect both payoffs and state transitions.

Observability

Players may not know the full state or the other players actions.

Strategy

Players must consider both current beliefs and future consequences.

Payoff

Can be immediate or discounted over time (infinite/finite horizon).

Example Scenarios

1.      Multi-agent reinforcement learning (MARL) in uncertain environments.

2.      Poker: A stochastic game with partial observability.

3.      Security patrolling: A defender and attacker in a grid-world; the attacker is only partially observable.

4.      Robot soccer: Teams compete in partially observable, probabilistic environments.

Solving or Approximating Solutions

Solving these games is complex due to:

         The combinatorial explosion of states/actions.

         The uncertainty from both stochasticity and partial observability.

Common approaches include:

         Dynamic programming (if full model is known).

         Monte Carlo Tree Search with belief updates.

         Bayesian Nash Equilibria (for games with incomplete info).

         Reinforcement learning (especially model-free methods).

         POMDP extensions to multi-agent settings (Dec-POMDPs).

Summary

Stochastic Observable Games are complex, multi-stage games with randomness and limited information. They model real-world situations like robot navigation, strategic military operations, or any competitive system under uncertainty.

Partially Observable Games

Partially Observable Games are strategic games where players do not have complete information about the game state, other players actions, or both. These are closer to real-world scenarios, where perfect knowledge is rare.

All game states that are indistinguishable to a player at a point in the game.

Something like the opponents hand in poker you dont know it, but it's critical.

Game Tree View

In fully observable games (like chess), each node in the game tree corresponds to a known, distinct state.

In partially observable games, players might face several nodes in the tree that look the same to them they must act based on the information set, not the true node.

Solving Partially Observable Games

There are two main approaches, depending on the setting:

1. Game-Theoretic Setting (Multi-Agent)

         Goal: Find Nash Equilibria or Bayesian Nash Equilibria.

         Model uncertainty using types (as in Bayesian games).

         Strategies are based on information sets, not just states.

         Common in competitive games like Poker or economic models.

2. Sequential Decision Making (Single/Multi-Agent)

         Goal: Maximize expected reward over time with partial info.

         Modeled as:

o    POMDP: Partially Observable Markov Decision Process (single-agent).

o    Dec-POMDP: Decentralized POMDP (multi-agent cooperative).

         Players maintain a belief state and plan accordingly.
